# Spec: ミニマル・クロマチックチューナー (iOS) — working name "Tone"

- Status: confirmed (v2 — review-with-codex 反映済み)
- Date: 2026-06-20
- Issue: なし (新規プロジェクト)

## Goal (1 文)
広告ゼロ・完全オフラインで、マイク許可済みなら起動 1 秒以内に楽器の音程ずれをセント単位で表示する、スイス派ミニマルデザインの iOS クロマチックチューナーを作る。

## 背景 / 動機
楽器チューナー市場はコモディティ化しており、コア機能(ピッチ検出)では差別化できない。実ユーザーの最大不満は「広告地獄・基本機能のペイウォール・bloatware」。本アプリは制作者のブランディング/キャリア成果物として、その対極(無広告・無課金・即起動・1 画面完結)を高い design craft で実装し、**App Store 公開実績 + OSS リポジトリ**の両輪でポートフォリオ価値を出す。収益化は目的ではない。差別化軸は**無広告 + ミニマルデザイン**。

## 設計判断 (mini-ADR)

### 判断 1: ピッチ検出方式 → AudioKit `PitchTap`
- A. AudioKit(SoundpipeAudioKit)`PitchTap` — 数十行・OSS標準 / アルゴリズムがブラックボックス
- B. 自前 YIN — 精度を詰められる・技術アピール / DSP実装コスト大
- C. CREPE(CNN) — 最高精度 / モバイル単音にオーバースペック
- **採用 A**。MVP最短 + デザイン軸へ工数集中。精度が課題化したら `PitchEngine` 境界を保ったまま B に差替可能。

### 判断 2: ピッチ検出を protocol で抽象化 → する
- A. `PitchEngine` protocol — 差替/モック注入可 / レイヤ1枚増
- B. ViewModel が直接 AudioKit — 最小コード / テスト不能
- **採用 A**。受け入れ基準を「合成 reading → 期待 Note/cents」で検証するためモック注入が必須。

### 判断 3: UI → SwiftUI (iOS 17+)
- A. SwiftUI — 宣言的・モダン / 低レベル描画は劣る
- B. UIKit + CoreAnimation — 描画自由度 / 記述量多
- **採用 A**。ミニマルUIに必要な要素は SwiftUI で十分。`@Observable` MVVM が組める。

### 判断 4: ドメインロジックを ViewModel から分離 → `TuningProcessor` に集約
- A. 平滑化/無音/Note変換/in-tune判定を純粋 `TuningProcessor`(値型)へ分離、ViewModel は状態保持と engine ライフサイクルのみ — テスト独立・SRP遵守 / 型が1つ増える
- B. すべて ViewModel に集約 — コード最小 / Fat ViewModel・テスト時に thread/time/UserDefaults に依存
- **採用 A**。受け入れ基準1〜12をUI・実機・時計から独立に単体テストするため、純ロジックを分離する。

### 判断 5: 時刻・永続化を protocol 注入 → する
- A. `Clock` / `ReferencePitchStore` を protocol 化し注入 — テストで test clock / in-memory store を使え、`sleep` とグローバル `UserDefaults` 依存を排除 / protocol 2つ増
- B. 直接 `Date`/`UserDefaults` — 最小 / 無音タイムアウトと永続化のテストが不安定
- **採用 A**。無音判定(時間依存)と永続化(AC11)を決定的にテストするため。

### 判断 6: ピッチ平滑化は対数(セント)空間 + オクターブ外れ値除去
- A. 検出周波数を `m = 1200·log2(f / refA4)`(refA4 からのセント)に写してから平滑化。オクターブ飛び(|Δm| > 600 cents の単発)は外れ値として棄却、連続2フレームで受理(ヒステリシス) — 音楽的に正しい / ロジックがやや複雑
- B. 周波数 Hz を線形 EMA — 単純 / **440と880の線形平均=660Hz≈E5 で音楽的に無意味**(v1の誤り)
- **採用 A**。Hz 線形平滑化は音高として破綻するため棄却。

## アーキテクチャ概要

```
View (SwiftUI)
  ↕ 観測
@MainActor TunerViewModel  ── engine ライフサイクル / state 保持 / setReferenceA4
  │  依存注入: PitchEngine, TuningProcessor, ReferencePitchStore, Clock
  ├─ PitchEngine (protocol) ──▶ AudioKitPitchEngine (SoundpipeAudioKit)  ※差替点
  ├─ TuningProcessor (値型/純) ── NoteConverter, 平滑化, 無音, in-tune
  ├─ ReferencePitchStore (protocol) ──▶ UserDefaultsReferencePitchStore
  └─ Clock (protocol) ──▶ MonotonicClock
```
依存方向は View → ViewModel → (protocol 群)。ドメイン層(`TuningProcessor`/`NoteConverter`/`ResolvedNote`)は SwiftUI にも AudioKit にも依存しない。物理 SPM ターゲット分割は任意(必須としない)。

## 公開 Interface

```swift
// ── ドメイン値型 ───────────────────────────────
struct PitchReading: Equatable {
    let frequency: Double       // Hz (PitchTap 検出値)。<=0/NaN/inf は無効入力として扱う
    let amplitude: Double       // PitchTap 生振幅。正規化レンジ保証なし(相対値) ※下記「振幅校正」
    let timestamp: TimeInterval // monotonic 秒 (Clock.now)
}

enum NoteName: String, CaseIterable { case C, Csharp, D, Dsharp, E, F, Fsharp, G, Gsharp, A, Asharp, B }

struct ResolvedNote: Equatable {
    let name: NoteName
    let octave: Int             // 科学的音高表記 (A4=440 で octave=4)
    let cents: Int              // -50...+50 (最寄り音からのずれ)
    let frequency: Double       // 入力周波数
}

// 周波数→音名 の純関数変換
struct NoteConverter {
    let referenceA4: Double      // 既定 440.0、有効範囲 415...466
    // アルゴリズム: f<=0 / !f.isFinite → nil。
    //   midi = round(69 + 12·log2(f / referenceA4))
    //   targetFreq = referenceA4 · 2^((midi-69)/12)
    //   cents = Int(round(1200·log2(f / targetFreq)))  // 定義上 -50...+50
    //   name = NoteName(midi % 12), octave = midi/12 - 1
    //   タイブレーク: ちょうど ±50 cents は round-half-up により上隣の音へ寄せる(一意)
    func note(for frequency: Double) -> ResolvedNote?
}

// ── 検出器の抽象化(差替/モック注入点)──────────
enum PermissionState: Equatable { case notDetermined, granted, denied }

enum PitchEngineError: Error, Equatable {
    case engineUnavailable   // AVAudioEngine 起動失敗 / media services reset
    case inputUnavailable    // 入力ルート無し / 他アプリが入力占有
}

protocol PitchEngine: AnyObject {
    // 配送は必ず main actor。実装は AudioKit コールバック(オーディオスレッド)を main へ marshaling する
    var onReading: (@MainActor (PitchReading) -> Void)? { get set }
    func requestPermission() async -> PermissionState   // 非同期(初回ダイアログ)
    func start() throws        // granted 前提で AVAudioSession 有効化 + tap 開始。冪等(二重 start は no-op)
    func stop()                // 冪等。stop 後は onReading を呼ばない
}

// ── 時刻・永続化の抽象(テスト注入)─────────────
protocol Clock { var now: TimeInterval { get } }                 // monotonic 秒
protocol ReferencePitchStore { func load() -> Double?; func save(_ hz: Double) }

// ── 純ロジック: 平滑化・無音・in-tune を集約 ────
struct TuningConfig {
    var emaAlpha: Double        = 0.2     // セント空間 EMA 係数
    var octaveRejectCents: Double = 600   // |Δ| 超で単発フレームを外れ値棄却
    var silenceTimeout: TimeInterval = 1.0
    var amplitudeGate: Double   = 0.02    // 相対振幅ゲート(下記「振幅校正」で実機調整)
    var inTuneCents: Int        = 3       // |cents|<=3 を in-tune
}

struct TuningState: Equatable {
    var smoothedCentsFromRef: Double?     // refA4 からの平滑化セント(連続量)
    var lastValidAt: TimeInterval?
    var note: ResolvedNote?
    var inTune: Bool
}

struct TuningProcessor {
    let converter: NoteConverter
    let config: TuningConfig
    // 1 reading を取り込み新 state を返す純関数。無効/低振幅/オクターブ外れ値の扱いを内包
    func ingest(_ state: TuningState, _ reading: PitchReading) -> TuningState
    // 無音タイムアウト評価。lastValidAt から silenceTimeout 超過なら note=nil(=無音)
    func evaluateSilence(_ state: TuningState, now: TimeInterval) -> TuningState
}

// ── 画面状態 ───────────────────────────────────
@MainActor @Observable
final class TunerViewModel {
    enum State: Equatable {
        case idle                                   // 起動直後
        case requestingPermission                   // 初回許可ダイアログ表示中
        case listening                              // 許可済み・音を拾えていない(無音)
        case tuning(note: ResolvedNote, inTune: Bool)
        case permissionDenied                       // 設定アプリ導線を表示
        case engineError(PitchEngineError)          // .engineUnavailable / .inputUnavailable で再試行導線
    }
    private(set) var state: State
    private(set) var referenceA4: Double

    init(engine: PitchEngine, processor: TuningProcessor,
         store: ReferencePitchStore, clock: Clock)

    func onAppear() async       // store.load → requestPermission → granted なら start。下記「起動シーケンス」
    func onDisappear()          // engine.stop()
    func setReferenceA4(_ hz: Double)  // 415...466 クランプ → store.save → processor/converter 更新
    func retry() async          // engineError からの再試行
}
```

## 挙動仕様

### 起動シーケンス (`onAppear`)
1. `store.load()` で基準ピッチ復元(無ければ 440)。
2. `engine.requestPermission()` を await。`.notDetermined` 中は `state = .requestingPermission`。
3. `.granted` → `engine.start()`。成功で `state = .listening`、`PitchEngineError` で `state = .engineError(...)`。
4. `.denied` → `state = .permissionDenied`。
- **「起動 1 秒以内」は許可済み(warm)起動にのみ適用**。初回起動は権限ダイアログ完了までを除外する(受け入れは手動・別ケース)。

### ピッチ処理 (`TuningProcessor`、main actor 上)
- `onReading` は `@MainActor` で配送される前提。ViewModel は受信 reading を `processor.ingest` に渡す。
- **振幅ゲート**: `reading.amplitude < config.amplitudeGate` のフレームは「無効」として state.note 更新に使わない(lastValidAt も更新しない)。
- **平滑化(対数空間)**: 有効フレームは `m = 1200·log2(f / referenceA4)` を計算。`smoothedCentsFromRef == nil` なら初期値 `m`。それ以外:
  - **オクターブ外れ値**: `abs(m - smoothed) > octaveRejectCents` かつ直前も外れ値でない(単発)なら **棄却**(state 不変)。2 フレーム連続で外れたら受理し EMA を新値へリセット(ヒステリシス)。
  - 受理時 `smoothed = emaAlpha·m + (1-emaAlpha)·smoothed`。
- 平滑化後 `f' = referenceA4 · 2^(smoothed/1200)` を `converter.note(for: f')` に通し `note` を得る。`inTune = abs(note.cents) <= inTuneCents`。`lastValidAt = reading.timestamp`。
- **無音判定**: `evaluateSilence(now:)` を UI 更新周期(タイマー/`TimelineView`)で呼び、`now - lastValidAt > silenceTimeout` なら `note=nil`(ViewModel は `.listening`)。`Clock` 注入で決定的にテスト可能。callback の有無に依存しない。

### 基準ピッチ
既定 A4=440Hz。設定 UI から 415〜466Hz を 1Hz 刻みで変更でき `ReferencePitchStore` に永続化、次回起動で復元。`setReferenceA4` は範囲外を端値にクランプ。

### マジックナンバーの根拠 / 振幅校正
- `inTuneCents=3`: スマホマイク前提では厳しめ。MVP は 3 で出し、実機評価で 3〜5 に調整可能な `TuningConfig` 値とする(ハードコードしない)。
- `amplitudeGate=0.02`: **`PitchTap` の amplitude は正規化レンジ未保証**。M2 の実機評価で静音時/演奏時の実測分布から再校正する。`TuningConfig` 経由で変更可能にし、固定値を仕様の前提にしない。

### AVAudioSession / 割り込み (`AudioKitPitchEngine` の責務)
- category = `.record`(再生不要)、mode = `.measurement`、`setActive(true)` を `start()` で、`setActive(false)` を `stop()` で。
- `AVAudioSession.interruptionNotification`(電話/Siri): began → tap 停止し `state` を `.engineError(.inputUnavailable)` 相当へ、ended(`.shouldResume`)→ 自動 `start()` 再開。
- `routeChangeNotification`(有線/Bluetooth マイク抜き差し): エンジン再構築。
- `mediaServicesWereResetNotification`: エンジン破棄→再生成。
- `requestPermission` は iOS 17+ の `AVAudioApplication.requestRecordPermission` を使用。

### ネットワーク/アカウント/解析
一切持たない。完全オフライン。第三者 SDK(広告・解析)を含めない。AudioKit/SoundpipeAudioKit の Privacy Manifest を確認し、アプリ側 App Privacy は「データ収集なし」。

## エッジケース

| 入力 / 状況 | 期待挙動 |
|---|---|
| `frequency <= 0` / `NaN` / `inf` | `NoteConverter.note` は `nil`、`TuningState` 不変 |
| `amplitude < gate`(無音/極小) | 無効フレーム。note/lastValidAt 更新せず、timeout 経過で `.listening` |
| 単発オクターブ飛び(440→880→440) | 880 を外れ値棄却。出力は A4 のまま揺れない |
| 連続オクターブ移動(880 が 2 フレーム継続) | 2 フレーム目で受理、A5 系へ追従 |
| 初回マイク権限 `.notDetermined` | `.requestingPermission` 表示、応答後に分岐 |
| 権限 `.denied` | `.permissionDenied`、設定アプリ導線 |
| 電話/Siri 割り込み | `.engineError(.inputUnavailable)`、ended で自動再開 |
| 他アプリが入力占有 / 入力ルート無し | `start()` が `.inputUnavailable` throw → `.engineError`、`retry()` で再試行 |
| AVAudioEngine 起動失敗 / media reset | `.engineError(.engineUnavailable)`、再構築 |
| `setReferenceA4(500)` / `(400)` | 466 / 415 にクランプ |
| バックグラウンド遷移(`scenePhase`) | `.background` で `stop()`、`.active` で `start()` 再開 |
| 極端な低音/高音(20Hz/5000Hz) | 計算は行うが帯域フィルタは MVP 対象外(out of scope) |

## 受け入れ基準

### 自動テスト(`NoteConverter` / `TuningProcessor` を純粋にテスト。engine/clock/store はモック)

要件 → 基準の対応:

| 要件 | AC |
|---|---|
| 音名変換 | AC1, AC3, AC5 |
| セント精度 | AC2 |
| 無効入力 | AC4 |
| 基準ピッチクランプ+永続化 | AC6, AC11 |
| 状態遷移(検出/無音) | AC7, AC8 |
| 権限フロー | AC9, AC13 |
| in-tune 境界 | AC10 |
| オクターブ外れ値除去 | AC12 |
| エラー状態分離 | AC14 |

1. `NoteConverter(referenceA4:440).note(for:440)` → `(.A, 4, 0, 440)`。
2. `note(for:446.16)` → `name=.A, octave=4, cents=24`(±1 許容、検算: +24.07c)。
3. `note(for:261.63)` → `(.C, 4, 0)`(検算: +0.03c→0)。
4. `note(for:0)` / `note(for:-10)` / `note(for:.nan)` / `note(for:.infinity)` → すべて `nil`。
5. `NoteConverter(referenceA4:442).note(for:442)` → `(.A, 4, 0)`。
6. `setReferenceA4(500)`→実効 466、`setReferenceA4(400)`→実効 415。
7. `processor.ingest(初期state, reading(freq:440, amp:0.5, t:0))` → `note=A4, inTune=true`。ViewModel 反映で `.tuning(A4, true)`。
8. 検出後、`evaluateSilence(now: lastValidAt + 1.01)` → `note=nil`。ViewModel 反映で `.listening`(test clock 注入、sleep 不使用)。
9. `engine.requestPermission()` がモックで `.denied` → `onAppear` 後 `state == .permissionDenied`。
10. `cents=3` の音 → `inTune=true`、`cents=4` → `false`(境界 `|cents|<=3`)。
11. `setReferenceA4(442)` 後に新 ViewModel を同一 store で生成 → `referenceA4==442`(永続化復元)。
12. `ingest` に `[440(t0), 880(t1), 440(t2)]`(各 amp 0.5)を順に与える → 3 出力すべて `note.name==.A, octave==4`(880 は単発オクターブ外れ値として棄却され、出力は揺れない)。
13. `requestPermission()` が `.granted` を返し `start()` 成功 → `state == .listening`(初回権限フロー、warm とは別ケース)。
14. `start()` が `.inputUnavailable` throw → `state == .engineError(.inputUnavailable)`。`.engineUnavailable` throw → `state == .engineError(.engineUnavailable)`(`.permissionDenied` と区別)。

### 手動受け入れ(M2・実機)
- **warm 起動**(許可済み)でチューニング表示まで 1 秒以内、スプラッシュ/ポップアップ/広告ゼロ。
- 初回起動: 権限ダイアログ → 許可 → 表示まで遷移が破綻しない(1 秒制約は適用外)。
- 1 画面に「音名・オクターブ・セントずれ・細い針(またはバー)・基準ピッチ」のみ。余白主体、モノトーン + 1 アクセント色。
- in-tune 時にアクセント色が変化、`abs(cents)` に比例して針が動く。
- VoiceOver で音名と「高い/低い/合っている」を読み上げ。Dynamic Type で数値が破綻しない。
- 実機シナリオ: fresh install allow/deny、background/foreground、電話/Siri 割り込み、Bluetooth/有線マイク route 変更、無音 1 秒復帰、A4 追従レイテンシ、App Privacy「収集なし」根拠確認。
