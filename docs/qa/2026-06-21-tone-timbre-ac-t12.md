# 手動 QA: 音叉モード音色選択 (AC-T12)

- 対象 spec: `docs/specs/2026-06-21-tone-timbre.md`
- 関連 PR: #11 (feat/tone-timbre)
- 種別: **手動・iOS 検証**(自動範囲外。AC-T1〜T11 は ToneCoreTests で自動担保済)
- 実施環境: iOS Simulator または実機(音声出力のため**実機推奨**)+ イヤホン/外部スピーカ

## 前提セットアップ

1. ビルドして起動 → 音叉モード(FORK)へ切替。
2. 音名 = A、オクターブ = A4(440Hz の基準音)に合わせる。
3. 音色チップ列が再生ボタン上に表示され、初期選択 = **サイン**(`.default`)であることを確認。

---

## チェックリスト

### (a) 4 音色が可聴・聞き分け可能

各チップを順に選択し再生する(停止せず切替でも可)。

- [ ] **サイン (sine)** — 純音。倍音なしの丸い音。
- [ ] **三角 (triangle)** — sine よりわずかに芯がある。
- [ ] **ノコギリ (sawtooth)** — 明らかにブザー的で倍音が多い。
- [ ] **音叉 (fork)** — FM 合成。減衰を伴う金属的/ベル的な響き。
- [ ] 隣接音色(sine↔triangle, triangle↔sawtooth)が**耳で区別できる**。

### (b) 切替時のクリック/無音ギャップなし

再生中に各チップを連続タップして音色を切り替える。

- [ ] 切替の瞬間に**クリック音(プチノイズ)が出ない**。
- [ ] 切替時に**無音ギャップ(音の途切れ)が知覚されない**。
- [ ] 同一音色を再選択しても音が途切れない(AC-T9 冪等性の聴感確認)。

### (c) fork が sine より温かい(緩い主観基準)

- [ ] fork が sine と比べて**倍音が多く「温かい/豊か」**に聞こえる(a/b より緩い基準)。

### (d) A4 実測ピッチ 440±2Hz

別のチューナーアプリ(または本アプリのチューナーモードを別端末で)で A4 を実測。

- [ ] **サイン** A4 = 440±2Hz。
- [ ] **三角** A4 = 440±2Hz。
- [ ] **ノコギリ** A4 = 440±2Hz。
- [ ] **音叉** A4 = 440±2Hz(FM のキャリア基本周波数が 440 に乗る)。

### (e) 音量正規化

4 音色を同条件(端末音量固定)で順に再生。

- [ ] 音色間の**知覚音量差が大きく崩れていない**(どれかが極端に大きい/小さいことがない)。

### VoiceOver / selected trait(a11y)

VoiceOver を ON にしてチップ列をスワイプ走査。

- [ ] 各チップが順に読み上げられる。読み上げラベル:
  - サインチップ → 「**サイン波**」
  - 三角チップ → 「**三角波**」
  - ノコギリチップ → 「**ノコギリ波**」
  - 音叉チップ → 「**音叉、FM**」
- [ ] **選択中**のチップに「**選択済み**」(`.isSelected` trait)が読み上げられる。
- [ ] 別チップを選択すると選択トレイトが移動する。

### Dynamic Type

- [ ] 文字サイズを最大付近に上げてもチップラベルが破綻せず操作可能。

---

## 判定

- 全項目 ✓ → AC-T12 **合格**。
- (a)(b)(d)(e) のいずれか不合格 → **要修正**(ToneAudio の波形生成/正規化/エンベロープを見直し)。
- (c) のみ不合格 → fork の FM パラメータ調整(緩い基準のため blocker ではない)。
